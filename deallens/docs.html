Quickstart

Go from zero to your first deal snapshot in under 5 minutes. This guide walks you through account setup, your first API call, and reading the results.

1. Get your API key

After your account is provisioned, generate an API key from the Settings → API Keys panel in the DealLens dashboard. Each key is scoped to a single workspace.

ℹ API keys start with dlk_. Keep them secret — treat them like passwords. You can rotate keys at any time from the dashboard.

2. Upload your first deck

Send a pitch deck (PDF or PPTX, max 50 MB) to the upload endpoint. The API returns a deal_id immediately and begins processing asynchronously.

bash
curl -X POST https://api.deallens.com/v1/decks \
  -H "Authorization: Bearer dlk_your_api_key" \
  -F "[email protected]"

# Response
{
  "deal_id": "deal_8x7k2m9p",
  "status": "processing",
  "created_at": "2026-05-27T14:30:00Z"
}

3. Retrieve the snapshot

Poll the snapshot endpoint or use webhooks (recommended) to get notified when analysis is complete. Processing typically takes 15–30 seconds.

bash
curl https://api.deallens.com/v1/deals/deal_8x7k2m9p/snapshot \
  -H "Authorization: Bearer dlk_your_api_key"

# Returns the full snapshot JSON (see Data Models)

4. Read the verdict

Every snapshot includes a verdict field with one of three values: Pass, Explore, or Fast-track. Use this as a starting signal — not a final decision. The verdict_reasoning field explains why.

✓ Use the SDK (Python or Node) instead of raw cURL for production integrations. See Installation below.

Installation & Setup

Install the DealLens SDK in your preferred language and configure your environment.

Python SDK

bash
pip install deallens
python
from deallens import DealLens

client = DealLens(api_key="dlk_your_api_key")

# Upload and analyze a deck
deal = client.decks.upload("pitch-deck.pdf")
print(deal.id)  # deal_8x7k2m9p

# Wait for the snapshot (blocks until ready)
snapshot = client.deals.get_snapshot(deal.id, wait=True)
print(snapshot.verdict)        # "Fast-track"
print(snapshot.company_name)   # "Synthera AI"
print(snapshot.red_flags)      # ["Revenue concentration...", ...]

Node.js SDK

bash
npm install @deallens/sdk
javascript
import { DealLens } from '@deallens/sdk';

const client = new DealLens({ apiKey: 'dlk_your_api_key' });

// Upload a deck
const deal = await client.decks.upload('./pitch-deck.pdf');

// Get the snapshot (waits for processing)
const snapshot = await client.deals.getSnapshot(deal.id, { wait: true });

console.log(snapshot.verdict);       // "Fast-track"
console.log(snapshot.companyName);   // "Synthera AI"

Environment Variables

Both SDKs automatically read the DEALLENS_API_KEY environment variable, so you don't need to pass it explicitly in code:

bash
export DEALLENS_API_KEY="dlk_your_api_key"
⚠ Never commit API keys to version control. Use .env files or your platform's secret manager.

Authentication

All API requests require a Bearer token in the Authorization header.

Include your API key in every request:

http
Authorization: Bearer dlk_your_api_key

Requests without a valid key return a 401 Unauthorized error. Keys can be scoped to read-only or full access from the dashboard.

Submit a pitch deck for analysis.

Get Snapshot

Retrieve the analyzed deal snapshot.

GET /v1/deals/{deal_id}/snapshot

Path Parameters

ParameterTypeDescription
deal_id requiredstringThe deal identifier returned from upload

Query Parameters

ParameterTypeDescription
waitbooleanIf true, the request blocks until analysis completes (max 60s timeout). Default: false.
sectionsstringComma-separated list of sections to include. Default: all. Options: overview,team,market,traction,competitors,financials,red_flags,questions,verdict

Response 200 OK

Returns the full snapshot object. See Snapshot Schema for the complete shape.

List Deals

Retrieve all deals in your workspace.

GET /v1/deals

Query Parameters

ParameterTypeDescription
statusstringFilter by status: processing, completed, failed
verdictstringFilter by verdict: pass, explore, fast-track
limitintegerMax results per page. Default: 25, max: 100.
cursorstringPagination cursor from previous response

Delete Deal

Permanently delete a deal and its snapshot.

DELETE /v1/deals/{deal_id}

Returns 204 No Content on success. This action is irreversible.

Webhooks

Get notified when deal analysis completes instead of polling.

Setup

Pass a webhook_url when uploading a deck, or configure a default webhook URL in Settings → Webhooks. DealLens sends a POST request with the following payload when processing completes:

json
{
  "event": "deal.snapshot.ready",
  "deal_id": "deal_8x7k2m9p",
  "verdict": "fast-track",
  "company_name": "Synthera AI",
  "completed_at": "2026-05-27T14:30:22Z",
  "snapshot_url": "https://api.deallens.com/v1/deals/deal_8x7k2m9p/snapshot"
}

Verification

All webhook requests include a X-DealLens-Signature header. Verify it using your webhook secret (found in the dashboard) to ensure the request is authentic.

Verdict Types

The three possible preliminary verdicts and what they signal.

VerdictSignalSuggested Action
PassSignificant structural concerns — weak team-market fit, red flags outweigh opportunity, or fundamentally flawed assumptions.Archive unless a partner has strong conviction. Document reasoning for LP reporting.
ExploreInteresting opportunity with open questions. Potential is there but needs more diligence before committing time.Schedule a 30-min intro call. Use the generated questions as your agenda.
Fast-trackStrong signal across team, market, and traction. Competitive deal dynamics likely.Flag to partners immediately. Schedule deep-dive within 1 week. Begin reference checks.
ℹ Verdicts are preliminary signals, not investment recommendations. Always apply your own judgment, domain expertise, and firm-specific criteria before making decisions.

Error Codes

Standard error responses returned by the API.

CodeStatusDescription
400Bad RequestInvalid file format, missing required fields, or file exceeds 50 MB.
401UnauthorizedMissing or invalid API key.
403ForbiddenAPI key does not have permission for this operation.
404Not FoundDeal ID does not exist in your workspace.
409ConflictDeal is still processing. Use ?wait=true or webhooks.
422UnprocessableFile was readable but contained insufficient content for analysis (e.g., blank slides, image-only deck with no extractable text).
429Rate LimitedToo many requests. Default limit: 60 requests/minute. Check Retry-After header.
500Server ErrorInternal error. Retry with exponential backoff. If persistent, contact support.
